# SpecForge Backend

The SpecForge Backend is a high-performance REST API built with Go, designed to handle complex specification modeling, validation, and governance logic.

## üöÄ Tech Stack

- **Language**: [Go 1.25+](https://go.dev/)
- **Web Framework**: [Echo v4](https://echo.labstack.com/)
- **Database**: [PostgreSQL 16+](https://www.postgresql.org/)
- **ORM/Query Builder**: [sqlc](https://sqlc.dev/) (Type-safe SQL)
- **Migrations**: [golang-migrate](https://github.com/golang-migrate/migrate)
- **Validation**: [gojsonschema](https://github.com/xeipuuv/gojsonschema)
- **Logging**: [Uber-go zap](https://github.com/uber-go/zap)
- **AI Integration**: [OpenAI Go SDK](https://github.com/sashabaranov/go-openai) & [Google Generative AI](https://github.com/google/generative-ai-go)

## üõ†Ô∏è Getting Started

### Prerequisites

- **Go**: 1.25+
- **PostgreSQL**: 16+
- **sqlc**: (Optional, for generating code from SQL)

### Installation

1.  Navigate to the backend directory:
    ```bash
    cd backend
    ```
2.  Download dependencies:
    ```bash
    go mod download
    ```

### Configuration

Create a `.env` file in the project root (or `backend/.env`) based on the `.env.example` provided in the root:

```env
DATABASE_URL=postgres://specforge:specforge@localhost:5432/specforge?sslmode=disable
JWT_SECRET=your-secret-key
ENVIRONMENT=development
PORT=8080
OPENAI_API_KEY=your-api-key
```

### Running the API

To start the server in development mode:

```bash
go run cmd/api/main.go
```

The API will be available at `http://localhost:8080`.

### Database Migrations

Migrations are located in `backend/migrations`. If you are using Docker, migrations run automatically. For local manual execution, use the `migrate` CLI:

```bash
migrate -path migrations/ -database "$DATABASE_URL" up
```

## üìÅ Project Structure

- `cmd/api`: Entry point for the REST API.
- `internal/app`: Core application logic and service layers.
- `internal/domain`: Domain models and interface definitions.
- `internal/repository`: Database access layer (generated by sqlc).
- `migrations`: SQL migration files.
- `pkg`: Shared utility packages.
